# Persian (Farsi)

Use formal literary Persian (فارسی ادبی). File suffix `_fa`. Persian pages must include `dir="rtl"` on the `<html>` tag: `<html lang="fa" dir="rtl">`. RTL-specific CSS overrides are in `style.css` using `[dir="rtl"]` selectors (shared with Arabic and Hebrew).

## Register

Elevated literary Persian — the register of Iranian academic and scholarly publishing. Formal but readable. Not colloquial spoken Farsi, not archaic Qajar-era prose. Think university press theological scholarship.

## Ezafe construction

Persian uses the ezafe (اضافه) construction for noun-noun and noun-adjective linkage. Ensure proper use throughout — this is one of the most distinctive features of Persian prose. The ezafe kasre (-e) is not written in standard Persian orthography but must be grammatically correct.

## Script notes

Persian uses the Arabic script with four additional letters: پ (pe), چ (che), ژ (zhe), گ (gaf). Ensure these are used correctly — agents sometimes substitute Arabic equivalents (ب for پ, ج for چ, etc.).

## No diacritics needed

Unlike Arabic, standard Persian text does not use tashkeel (vowel diacritics). Do not add harakat to Persian text. The exceptions are:
- Occasional kasre for disambiguation in rare cases
- Short vowel marks on Arabic loanwords only when ambiguity would result

## Psalm numbering

Keep **Vulgate numbering** (matching the source commentary and all other language versions on the site). Persian Bibles typically follow the Masoretic/Hebrew numbering, so add the modern number in parentheses on first occurrence or in verse/section headings — e.g., «مزمور ۸۹ (۹۰ در شماره‌گذاری عبری)».

## Numerals

Use Persian/Eastern Arabic numerals (۰۱۲۳۴۵۶۷۸۹) in body text, not Western Arabic numerals (0123456789). This follows standard Persian typographic convention.

## Agent pitfalls

- **Arabic substitution**: Agents may produce Arabic text instead of Persian, or use Arabic letter forms (ك instead of Persian ک, ي instead of Persian ی). Persian uses ک (kaf with short vertical stroke) and ی (yeh without dots below). This is the most critical and common error.
- **Colloquial register**: Agents may default to spoken Farsi (e.g., می‌خوام instead of می‌خواهم). Insist on formal literary register.
- **Western numerals**: Agents default to 0-9. Must explicitly instruct to use ۰-۹.
- **Missing half-space (نیم‌فاصله)**: Persian uses a zero-width non-joiner (ZWNJ / half-space) between prefix/suffix and stem in compound verbs and affixed words (e.g., می‌کند not میکند, کتاب‌ها not کتابها). Agents frequently omit these.
- **Ezafe errors**: Agents may construct phrases with incorrect or missing ezafe linkage.
- **Biblical names**: Agents may use Arabic forms instead of Persian forms (e.g., عيسى instead of عیسی, داود instead of داوود).
